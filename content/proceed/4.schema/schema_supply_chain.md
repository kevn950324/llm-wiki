# Schema：供應鏈頁 (lib/3.supply_chain/)

> **定位**：以**具體客戶鏈 / 平台鏈 / 主題鏈**為主軸建立供應鏈地圖。
> - ✅ `供應鏈_AWS_Trainium3`、`供應鏈_CoWoS`、`供應鏈_光通訊`、`供應鏈_低軌衛星`
> - ❌ `供應鏈_封測`、`供應鏈_交換器`、`供應鏈_PCB材料`、`供應鏈_機殼`（這些是公司角色或產品環節，應使用 `#環節/*`）
>
> 建立時機：使用者明確要求，或 ingest 偵測到後詢問確認。

## 視覺化規範

每個供應鏈頁至少要有：
1. **文字頁** `供應鏈_XXX.md` — 廠商列表、競爭格局、觀察重點（wikilink 連結）
2. **結構圖** — 二擇一：
   - **優先**：`供應鏈_XXX.canvas`（Obsidian Canvas 圖譜，呼叫 `obsidian-visual-skills:obsidian-canvas-creator` skill）
   - **可選**：在文字頁內嵌 Mermaid `flowchart`（節點少於 10 個、或快速查詢用）

兩者皆無時，lint 會回報缺圖。除非使用者明確要求 Excalidraw，或維護既有 `.flowchart.md`，否則新圖不使用 Excalidraw。

**顏色規範**（Canvas 與 Mermaid `classDef` 共用，與 `proceed/5.skill/visualize/visualize.md` 一致）：

| 環節 | 顏色 | hex |
|------|------|-----|
| 上游設備 | 淺紅 | `#ffc9c9` |
| 製程環節 | 淺紫 | `#d0bfff` |
| 外部材料 | 淺橙 | `#ffd8a8` |
| 核心廠商 | 淺藍 | `#a5d8ff` |
| 終端客戶 | 淺黃 | `#fff3bf` |
| HBM / 記憶體 | 淺青 | `#c3fae8` |
| 基板 / 材料 | 淺綠 | `#b2f2bb` |

**連線規範：**
- 製程流向 / 供應鏈主線：使用有向 edge
- 材料投入 / 補充關係：可使用有向 edge 並加簡短 label
- 避免同一節點間建立重複或語意相近的連線
- Canvas 節點文字保持簡短；詳細論點寫回文字頁

**參考範例：** `[[供應鏈_CoWoS.canvas]]`

## 注意事項
- `title` 以 `供應鏈_` 開頭
- 每個供應鏈頁需要有結構圖：**優先 Canvas，可用 Mermaid**（依複雜度與使用情境）
- Canvas 節點用顏色區分不同環節；Mermaid 用 `classDef` 呼應顏色規範
- 規模大、跨環節多時建議用 Canvas（互動性高）；簡單線性鏈可直接 Mermaid
