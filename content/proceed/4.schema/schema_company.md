# Schema：公司頁 (lib/1.company/)

## 目錄結構

```
lib/1.company/
├── TW/           台股（按代號數字由小到大排列）
├── Overseas/     國外股票
└── Unlisted/     未上市公司、子公司、CSP 內部單位
```

## 命名規則

> **重要**：`檔名`、frontmatter `title` 欄位、H1 標題三者必須完全一致。

### 台股（lib/1.company/TW/）

**格式**：`{股票代號}_{公司名稱}（上市別）`

| 上市別 | 說明 | 括號用字 | 範例 |
|--------|------|---------|------|
| TWSE 上市 | 台灣證券交易所 | `（市）` | `2330_台積電（市）` |
| TPEX 上櫃 | 櫃買中心 | `（櫃）` | `4749_新應材（櫃）` |
| 興櫃 | 興櫃市場 | `（興）` | `3081_聯亞（興）` |
| 未上市/未上櫃 | 無代號，代號欄留空 | `（未）` | `光陽光電（未）` |

- 有代號的公司依代號**數字由小到大**排列（目錄排序與 index 排序皆依此）
- 無代號（未上市櫃）排在有代號公司之後，按名稱筆畫排列

> [!warning] 上市別必須查證，不可憑代號範圍推測
> 台灣股票代號範圍與掛牌市場無對應規律（4xxx、7xxx 等皆可能是上市或上櫃）。
> 新增公司頁時須確認實際掛牌市場（Gemini 查詢）。
> 興櫃公司轉板後（→上市/上櫃）需同步更新檔名、title、H1 及全 vault wikilink。

### 國外股票（lib/1.company/Overseas/）

**格式**：`{代號}.{交易所代碼}({公司英文名小寫})`

| 交易所 | 代碼 | 範例 |
|--------|------|------|
| 美國（NYSE/NASDAQ） | `US` | `NVDA.US(nvidia)` |
| 荷蘭 | `NL` | `ASML.NL(asml)` |
| 日本 | `JP` | `6857.JP(advantest)` |
| 韓國 | `KR` | `005930.KR(samsung)` |
| 香港 | `HK` | `0700.HK(tencent)` |

### 未上市 / 子公司 / CSP 內部單位（lib/1.company/Unlisted/）

**格式**：`{公司名}（未）`

適用於無公開交易代號的供應商、客戶、子公司、CSP 內部晶片團隊或尚未建立正式上市資訊的公司實體。

| 類型 | 範例 |
|------|------|
| 未上市公司 | `Annapurna_Labs（未）` |
| CSP 內部單位 | `AWS（未）` |
| 子公司 / 事業體 | `Google_TPU（未）` |

若後續確認有公開上市母公司，仍可保留 Unlisted 頁作為業務單位頁，並在 `related_companies` 連到母公司頁。

## Frontmatter 規範

### 台股範例

```yaml
---
title: "2330_台積電（市）"
ticker: "2330.TW"          # 台股加 .TW；美股直接用代號（如 "NVDA"）
market: TW
exchange: TWSE             # TWSE / TPEX / 興櫃 / 未上市
sector: 晶圓代工
tags:
  - 公司/台積電
  - 技術/2nm
  - 供應鏈/CoWoS            # 具體研究主軸；公司角色用 #環節/*
  - 環節/晶圓代工
  - 產業/半導體
updated: 2026-05-03
aliases:
  - TSMC
  - 台積
related_companies:
  - "[[NVDA.US(nvidia)]]"       # 客戶 / 下游
  - "[[ASML.NL(asml)]]"         # 上游設備
  - "[[3711_日月光投控（市）]]" # 封測 / 先進封裝夥伴
---
```

### 國外範例

```yaml
---
title: "NVDA.US(nvidia)"
ticker: "NVDA"             # 美股直接用代號，不加交易所後綴
market: US
exchange: NASDAQ
sector: AI晶片
tags:
  - 公司/NVIDIA
  - 技術/GPU
  - 產業/AI伺服器
updated: 2026-05-03
aliases:
  - NVIDIA
  - 輝達
related_companies:
  - "[[2330_台積電（市）]]"     # 晶圓代工 / CoWoS
  - "[[6669_緯穎（市）]]"       # AI 伺服器 ODM

```

### Unlisted 範例

```yaml
---
title: "AWS（未）"
ticker: ""
market: Unlisted
exchange: 未上市
sector: 雲端 / AI 加速器
tags:
  - 公司/AWS
  - 產業/雲端
  - 產業/AI伺服器
updated: 2026-05-07
aliases:
  - Amazon Web Services
related_companies:
  - "[[3661_世芯-KY（市）]]"
  - "[[2330_台積電（市）]]"
---
```

### related_companies 規則

公司頁若資料中出現明確上下游、客戶、供應商、製造夥伴、通路/代工、競爭或替代關係，必須同步更新 frontmatter `related_companies`。

若目前沒有明確公司關係、只有產業同業比較、或關係尚待查證，`related_companies: []` 合法；不要為了填滿欄位硬塞不確定關係。

| 關係類型 | 範例 | 寫入方式 |
|----------|------|----------|
| 上游 | 設備、材料、零組件供應商 | 加入上游公司 wikilink |
| 下游 / 客戶 | CSP、品牌客戶、系統廠、終端應用客戶 | 加入下游或客戶公司 wikilink |
| 製造 / 代工夥伴 | 晶圓代工、封測、ODM、EMS | 加入合作或製造角色公司 wikilink |
| 同業 / 替代 | 競爭者、替代供應商、第二來源 | 加入競爭或替代公司 wikilink |

`related_companies` 只放 wikilink 字串，不在 YAML 內寫長敘述；關係說明寫在正文「供應鏈位置」或「相關公司」段落。更新既有公司頁時，必須保留既有 `related_companies`，只追加缺漏項目。

## 注意事項
- 台股 title 格式：有代號用 `{代號}_{名稱}（市/櫃/興）`，無代號用 `{名稱}（未）`
- 國外 title 格式：`{代號}.{交易所}({英文名})`，英文名全小寫
- 未上市、子公司、CSP 內部單位放 `lib/1.company/Unlisted/`，title 用 `{名稱}（未）`
- 每個公司頁**必須**有 `[[供應鏈_XXX]]` 連結
- Unlisted 頁若不是供應鏈參與者，可免除 `[[供應鏈_XXX]]` 必填，但需在基本資料說明定位
- EPS、目標價、產能、量產時程、供應商地位等關鍵 claim 需保留來源、來源日期、claim 類型與信心水準；查證流程見 `proceed/5.skill/source_verify/source_verify.md`
- 新建公司頁時可先建立基本頁，但基本資料不可過度簡略；至少補主要產品/服務、應用場景、供應鏈位置、已知客戶/上下游與資料來源
- 公司頁內容密度可比一行摘要更高：核心技術/競爭優勢、產品與應用、供應鏈位置、相關公司、催化劑、財務/EPS/目標價依資料可得性逐步補齊
- 若來源有產品圖、製程圖、供應鏈圖、產能圖、客戶/平台架構圖，應在公司頁或相關技術/供應鏈頁嵌入圖片並加簡短圖說；沒有來源圖片時可用 canvas圖補結構
- 重複或高度相似的投資論點需統一整理，不要按報告逐條堆疊造成畫面混亂
- 券商 EPS 預估使用年度 x 券商矩陣表，並保留報告發布日
- 目標價與評等獨立成表，保留報告發布日、評價基礎與來源
- 若資料揭露上下游、客戶、供應商、代工/製造夥伴、競爭或替代關係，必須同步補 `related_companies` 與正文「相關公司」/「供應鏈位置」說明
- 論點或數字衝突時先詢問；若保留多方說法，使用 `[!warning] 資訊衝突` callout
