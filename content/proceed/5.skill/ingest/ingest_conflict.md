# Ingest：衝突資訊處理

> 由 `ingest.md` 在偵測到資料衝突時引用。常規 ingest 流程不需要載入本檔。

若新資料與現有頁面矛盾：

- 先判斷是否為會影響結論的重大衝突（例如 EPS、目標價、產能、量產時間、供應商地位、投資論點方向）。
- 重大衝突需主動詢問使用者何者正確，避免混淆。
- 若可同時保留，必須用 `[!warning] 資訊衝突` callout 標示各來源、報告發布日與差異，不可把不同說法混寫成單一結論。
- 券商 EPS、目標價、評等衝突通常不算事實錯誤，應以券商別並列表保留。

## 標準格式

```markdown
> [!warning] 資訊衝突
> - [[報告_XXX]]（報告日：YYYY-MM-DD）：說法 A
> - [[報告_YYY]]（報告日：YYYY-MM-DD）：說法 B
> - 狀態：待使用者確認 / 多方預估並存
```

## 處理決策樹

| 情境 | 處理 |
|------|------|
| 兩份券商 EPS 估值不同 | 表格並列保留，標明來源券商與報告日 |
| 新報告與舊報告事實衝突（例：客戶名單、產能）| 詢問使用者，確認後保留正確值 + callout 紀錄變動 |
| 公司公告 vs 媒體報導 | 公告為準，媒體列為輔助 |
| Memo 與報告矛盾 | 詢問使用者；通常 memo 較新但需查證 |
| 推論 vs 公告事實 | 事實覆蓋推論，但保留推論作為 thesis 紀錄 |

衝突一律進 log 標明，避免後續 ingest 再次踩到同個矛盾。
